Урок по CROSS JOIN в PostgreSQL

Команда CROSS JOIN используется для выполнения декартового произведения двух таблиц. Это означает, что каждая строка из первой таблицы соединяется с каждой строкой из второй таблицы.

1. Базовый синтаксис

Синтаксис команды CROSS JOIN:


SELECT "столбцы" FROM имя_схемы."таблица1"
CROSS JOIN имя_схемы."таблица2";

Пример: предположим, у нас есть таблицы Showtimes (Показы) и Halls (Залы). Мы хотим получить все возможные комбинации Показов и залов:


SELECT * FROM public."Showtimes"
CROSS JOIN public."Halls";

2. Особенности использования CROSS JOIN

  • CROSS JOIN возвращает все возможные комбинации строк из двух таблиц.
  • Если первая таблица содержит N строк, а вторая таблица — M, результат будет содержать N × M строк.
  • Обычно CROSS JOIN применяется в ситуациях, где декартовое произведение данных действительно нужно, например, для генерации всех комбинаций значений.

Заключение

Команда CROSS JOIN полезна для создания декартового произведения двух таблиц. Её использование требует осторожности, так как результат может быть очень большим, особенно если таблицы содержат много строк.
