Урок по LEFT JOIN в PostgreSQL

Команда LEFT JOIN используется для соединения строк из двух таблиц. Она возвращает все строки из левой таблицы (основной), даже если в правой таблице (дополнительной) нет совпадений. Если совпадений нет, столбцы из правой таблицы заполняются значением NULL.

1. Базовый синтаксис

Синтаксис команды LEFT JOIN:


SELECT "столбцы" FROM имя_схемы."таблица1" идентификатор1
LEFT JOIN имя_схемы."таблица2" идентификатор2 ON идентификатор1."столбец" = идентификатор2."столбец";

Пример: предположим, у нас есть таблицы Hals (Залы) и Showtimes (Показы). Мы хотим получить расписание залов, включая те, которые не указаны в показах:


SELECT H.*, "ID_Movie", "Start_Time" FROM public."Halls" H
LEFT JOIN public."Showtimes" S ON S."ID_Hall" = H."ID_Hall";

2. Особенности использования LEFT JOIN

  • Все строки из левой таблицы всегда включаются в результат.
  • Если в правой таблице нет совпадений, столбцы из правой таблицы заполняются NULL.
  • LEFT JOIN часто используется для анализа данных, где важны все записи из основной таблицы.

Заключение

Команда LEFT JOIN полезна для получения всех строк из основной таблицы с дополнительной информацией из связанной таблицы. Даже если совпадений нет, данные из основной таблицы сохраняются в результатах.
