Урок по команде WHERE в PostgreSQL

Команда WHERE используется для фильтрации данных в запросах. Она позволяет выбрать только те строки, которые соответствуют заданному условию. В PostgreSQL WHERE может работать с различными операторами и выражениями для более точной фильтрации данных.

1. Синтаксис команды WHERE

Основной синтаксис команды WHERE выглядит следующим образом:

SELECT "столбцы" FROM схема."таблица" WHERE условие;

Где:

  • столбцы — это имена столбцов, которые вы хотите получить.
  • таблица — это имя схемы, где находится таблица.
  • таблица — это имя таблицы, из которой будет выполняться запрос.
  • условие — это логическое условие для фильтрации данных.

2. Пример использования WHERE

Пример 1: Фильтрация по году издания

Допустим, мы хотим выбрать все фильмы, выпущеные в жанре "Боевик". Для этого мы используем команду WHERE с условием:

SELECT * FROM public."Movies" WHERE "Genre" = 'Боевик';

Этот запрос вернёт все фильмы в жанре "Боевик".

Пример 2: Фильтрация по году выпуска

Теперь, допустим, мы хотим найти все фильмы, выпущенные после 2016. Для этого мы указываем в запросе условие для фильтрации по году выпуска:

SELECT * FROM public."Movies" WHERE "Release_Year" > 2016;

Этот запрос вернёт все фильмы, которые выпущенные после 2016.

Пример 3: Комбинированные условия

Можно комбинировать несколько условий с помощью оператора AND или OR. Например, чтобы выбрать фильмы, которые были изданы после 2016 года и принадлежат к жанру "Фантастика", используем:

SELECT * FROM public."Movies" WHERE "Release_Year" > 2016 AND "Genre" = 'Фантастика';

Этот запрос вернёт фильмы, которые одновременно соответствуют обоим условиям: изданы после 2016 года и принадлежат к жанру "Фантастика".

Заключение

Команда WHERE в PostgreSQL — это мощный инструмент для фильтрации данных. Она позволяет применять различные логические операторы и функции для точного поиска нужной информации. Используя условия с операторами AND, OR, LIKE, IN и другими, вы можете строить гибкие и эффективные запросы для работы с данными.
