Урок по BETWEEN, IN и NOT IN в PostgreSQL

В PostgreSQL команды BETWEEN, IN и NOT IN используются для фильтрации данных. Эти команды позволяют упрощать запросы и делать их более понятными.

1. Команда BETWEEN

Команда BETWEEN используется для проверки, входит ли значение в указанный диапазон (включительно).

Синтаксис:

SELECT "столбцы" FROM имя_схемы."имя_таблицы" WHERE "столбец" BETWEEN значение1 AND значение2;

Пример: найдём все фильмы, опубликованные с 2010 по 2018 год:

SELECT * FROM public."Movies" WHERE "Release_Year" BETWEEN 2010 AND 2018;

2. Команда IN

Команда IN используется для проверки, совпадает ли значение с одним из перечисленных.

Синтаксис:

SELECT столбцы FROM имя_таблицы WHERE столбец IN (значение1, значение2, значение3);

Пример: найдём все книги жанров "Боевик" или "Анимация":

SELECT * FROM public."Movies" WHERE "Genre" IN ('Боевик', 'Анимация');

3. Команда NOT IN

Команда NOT IN используется для исключения значений из перечисленных.

Синтаксис:

SELECT столбцы FROM имя_таблицы WHERE столбец NOT IN (значение1, значение2, значение3);

Пример: найдём все фильмы, которые не относятся к жанрам "Фантастика" или "Детектив":

SELECT * FROM public."Movies" WHERE "Genre" NOT IN ('Фантастика', 'Детектив');

4. Сравнение BETWEEN, IN и NOT IN

Эти команды служат для различных целей:

  • BETWEEN удобен для работы с диапазонами значений (например, годов).
  • IN удобен, когда нужно проверить совпадение со списком значений.
  • NOT IN позволяет исключить определённые значения из выборки.

Заключение

Команды BETWEEN, IN и NOT IN позволяют упростить запросы, делая их более выразительными и удобными для чтения. Их использование повышает гибкость работы с базой данных PostgreSQL.
