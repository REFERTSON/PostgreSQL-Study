Урок по LIMIT и OFFSET в PostgreSQL

Команды LIMIT и OFFSET используются для ограничения количества возвращаемых строк и управления выборкой данных в PostgreSQL. Они особенно полезны при постраничном выводе данных.

1. LIMIT

Команда LIMIT задаёт максимальное количество строк, которые должны быть возвращены запросом.

Синтаксис:

SELECT "столбцы" FROM имя_схемы."имя_таблицы" LIMIT количество_строк;

Пример: выберем первые 5 фильмов из таблицы Movies:

SELECT * FROM public."Movies" LIMIT 5;

Этот запрос вернёт только 5 строк из таблицы.

2. OFFSET

Команда OFFSET пропускает заданное количество строк перед началом возврата данных.

Синтаксис:

SELECT "столбцы" FROM имя_схемы."имя_таблицы" OFFSET количество_строк;

Пример: пропустим первые 5 строк и выведем все последующие:

SELECT * FROM public."Movies" OFFSET 5;

3. Комбинация LIMIT и OFFSET

Вы можете комбинировать LIMIT и OFFSET, чтобы вывести данные с определённого места в таблице. Это полезно для реализации постраничного вывода данных.

Синтаксис:

SELECT "столбцы" FROM имя_схемы."имя_таблицы" LIMIT количество_строк OFFSET смещение;

Пример: выберем 5 фильмов, начиная с шестого:

SELECT * FROM public."Movies" LIMIT 5 OFFSET 5;

Заключение

Команды LIMIT и OFFSET предоставляют удобные способы управления количеством строк, возвращаемых запросом. Они особенно полезны при работе с большими таблицами или создании интерфейсов постраничного просмотра данных.
