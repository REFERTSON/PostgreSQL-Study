Урок по INNER JOIN в PostgreSQL

Команда INNER JOIN используется для соединения строк из двух таблиц на основе совпадающих значений в указанных столбцах. Если значения совпадают, строки включаются в результат.

1. Базовый синтаксис

Синтаксис команды INNER JOIN:


SELECT "столбцы" FROM имя_схемы."таблица1" идентификатор1
INNER JOIN имя_схемы."таблица2" идентификатор2 ON идентификатор1."столбец" = идентификатор2."столбец";

Пример: предположим, у нас есть таблицы Movies (Фильмы) и Showtimes (Показы). Таблица Movies содержит информацию о фильмах, а Showtimes — о покуазах. Мы хотим получить расписание фильмов:


SELECT "Title", "Genre", "Start_Time", "ID_Hall" FROM public."Movies" M
INNER JOIN public."Showtimes" S ON S."ID_Movie" = M."ID_Movie";

2. Особенности использования INNER JOIN

  • Только строки с совпадающими значениями в указанных столбцах обеих таблиц включаются в результат.
  • Если совпадений нет, строка из обеих таблиц не будет включена в результат.
  • Вы можете использовать INNER JOIN с несколькими таблицами, добавляя новые соединения.

Заключение

Команда INNER JOIN является основным инструментом для соединения таблиц в PostgreSQL. Она позволяет объединять данные из связанных таблиц, основываясь на совпадении значений в определённых столбцах.
