Урок по UNION в PostgreSQL

Команда UNION используется для объединения результатов двух или более запросов. Она удаляет дублирующиеся строки, возвращая только уникальные результаты. Если нужно сохранить дубли, используется UNION ALL.

1. Базовый синтаксис

Синтаксис команды UNION:

    
SELECT "столбцы" FROM имя_схемы."имя_таблица1"
UNION
SELECT "столбцы" FROM имя_схемы."имя_таблица2";

Пример: предположим, у нас есть таблица Tickets (Билеты), и мы хотим объединить список покупателей, которые купили билеты, с покупателями, указанными в таблице Customers:


SELECT "ID_Customer" FROM public."Tickets"
UNION
SELECT "ID_Customer" FROM public."Customers";

2. Использование UNION ALL

Если нужно сохранить дублирующиеся строки, используется UNION ALL. Например:


SELECT author FROM books
UNION ALL
SELECT author FROM popular_authors;

4. Особенности использования UNION

  • Столбцы, возвращаемые запросами, должны иметь одинаковый тип данных и порядок.
  • Количество столбцов в каждом запросе должно совпадать.
  • Для упорядочивания объединённых данных можно использовать ORDER BY, но только после завершения всех запросов UNION.

Заключение

Команда UNION позволяет объединять результаты нескольких запросов, удаляя дубликаты. Для сохранения дубликатов используйте UNION ALL. Это удобный инструмент для объединения данных из разных источников в PostgreSQL.
